**Root cause (strong):** `tui/status.go::Clear()` sets `s.cleared=true`; `Show()` no-ops while `cleared` is true; `Reset()` is called only at turn start, so the 2nd tool's `Show()` in the same turn is ignored.

48. Read `tui/status.go` (Show/Clear/Reset, L75–140), and grep all callers of `StatusMsg.Show/Clear/Reset` (likely `tui/model.go`, `tui/tool_execution.go`).
49. Identify the event that clears the spinner between tools (interim clear on tool end). Confirm with a 2-tool repro: log `spinning`/`cleared` transitions.
50. Separate the two guards: keep a **session-ended** guard (`sessionEnded bool`) armed ONLY by the session-end handler (`handleSessionEnd`); remove the broad `cleared` no-op for in-turn transitions. Concretely: `Show()` should be allowed to restart after a normal in-turn `Clear()`; it must still refuse only when `sessionEnded` is true.
51. Add `StatusMsg.SessionEnd()` that sets `sessionEnded=true` and clears; call it from `handleSessionEnd`. `Reset()` (turn start) clears `sessionEnded` so a brand-new turn can spin.
52. Tests (`tui/status_test.go`):

    - `TestStatus_SessionEnd_BlocksLateShow`: SessionEnd→Show is a no-op.
    - `TestStatus_NewTurn_Reset_AllowsShow`: after SessionEnd, Reset() re-enables Show.
