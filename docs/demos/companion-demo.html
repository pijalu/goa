⟡ How to Use the Goa Companion

Sub-agent review in two modes: agent-driven and framework-driven

Demo: enabling framework-driven companion and asking it to write a tic-tac-toe game

Overview

The companion is a dedicated review sub-agent. It can be triggered by the main agent (agent-driven) or automatically after every turn (framework-driven).

When to use: When you want a second opinion on every change or when the agent itself decides to ask for review.

Step-by-Step

  1. Check companion status
    /companion
  2. Enable framework-driven mode
    Every main-agent turn is reviewed automatically:
    /companion:framework
  3. Send a request
    The main agent acts, then the companion reviews its output:
    write a html tic-tac-toe game
  4. Disable when done
    /companion:off

What Gets Created

  • tic-tac-toe.html — the requested game

Modes

/companion:agent # main agent chooses when to ask
/companion:framework # review after every turn
/companion:off # no companion
