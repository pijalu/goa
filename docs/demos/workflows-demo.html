⟡ How to Use Goa Workflows

Multi-stage, multi-agent pipelines (plan → coder → review)

Demo: running the built-in implement-feature workflow to create a tic-tac-toe game

Overview

Workflows define sequential stages. Each stage has a role and a prompt. The implement-feature workflow runs Plan → Implement → Review.

When to use: When you want structured execution with planning before coding and review after.

Step-by-Step

  1. List workflows
    /workflows:list
  2. Inspect a workflow
    /workflows:show implement-feature
  3. Run the workflow
    /workflows:implement-feature write a html tic-tac-toe game
  4. Review results
    Each stage produces output and the final file is written.

What Gets Created

  • index.html — landing page written by the planner
  • tic-tac-toe.html — the game written by the coder

Workflow Stages

Plan # decides structure and approach
Implement # writes the code
Review # checks the result
