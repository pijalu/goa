⟡ How to Use the Goa Orchestrator

Flexible multi-agent orchestration with hub, fanout, and pipeline topologies

Demo: a hub-topology run that delegates a tic-tac-toe game to a coder specialist

Overview

The orchestrator creates a run with one or more specialist agents. In this demo we use a hub topology: the orchestrator plans and delegates, and the coder specialist implements the file.

When to use: Complex tasks that benefit from decomposition, parallel exploration, or sequential specialist stages.

Step-by-Step

  1. Create a hub run
    Give the run a name and objective:
    /orchestrate:new:topology=hub,name=tictactoe,objective=write a html tic-tac-toe game
  2. Watch the orchestrator
    The orchestrator delegates to the coder, and the TUI shows each agent's tab and activity.
  3. Review the result
    When the coder writes the file, the run is complete.

What Gets Created

  • tic-tac-toe.html — the delegated file

Other Topologies

/orchestrate:new:topology=fanout,name=analysis,objective=... # parallel specialists
/orchestrate:new:topology=pipeline,name=build,objective=... # sequential stages

Resume a Run

/orchestrate:list
/orchestrate:resume:<run-id>

Transcript

[orchestrator] I'll delegate the implementation to a coder specialist. The objective is to write a single self-contained HTML file named tic-tac-toe.html.

[orchestrator] → delegate to coder

[coder] ■ written — 7,851 bytes, 276 lines
